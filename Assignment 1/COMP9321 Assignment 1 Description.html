COMP9321 Semester 1, 2015

Assignment 1: Auction-Item Data Store

Aims and Background

This project aims to give students hands-on experience in designing and implementing a functional Web-Application on their own. The theme of assignment will be oriented around action sites, similar to eBay. Such auction sites have revolutionised consumer-to-consumer business by allowing ordinary people to easily trade with each other.

This assignment will act as a kick-starter to Assignment 2. In the subsequent assignment you will be expected to design and implement an auction system capable of allowing users to offer items for auctions and participate in competitive bidding. It will also contain features such as user-registration and some admin functionality. However, in this assignment, you will only focus on the data-aspects of the "auction items", and implement the associated fundamental features. This component can then be integrated into your Assignment 2.

Requirements

1. Auction-Item Data Model

An auction-item in your application will need to conform to the provided schema. A sample AuctionItem can also be found at the following XML file. The database of your application will thus be in the form of an XML Database, and you will also therefore be required to apply XML parsing to manipulate the XML file.

You will associate data such as search preferences and a "Wish List" with a user using cookies and HTTP sessions. You should not use a login (user/password) for this purpose.

2. The Homepage

The starting/welcome page of your application, should display at the minimum a search bar allowing users to search for auction-items. Additionally, you should also provide to the users the ability to perform advanced search based on more specific search preferences.

The welcome page should also display a list (approx. 10 items) of the top-rated auction-items chosen at random.

3. Search

The search should be implemented to apply to keywords (word search including substrings) in the title, category or description of an auction-item. Clicking on search button should present the user with a list of auction-items that match the search criteria. On the search listing pages, each thumbnail of the search results should contain: (a) The title of the item; (b) A Picture of the item; and (c) Add to WishList button.

The title should act as a hyperlink that directs to another page with more complete details about the auction-item.

The WishList stores a collection of items. (Note: For now this should all be maintained via cookies/session attributes. You are not required to persist user data to the database. This will be part of the next assignment.)

The header of the search results page should also contain the search bar again, and associated options. The allows the user to conduct another search (cf. Google, etc.).

4. Add a 'new' Auction-Item

At the header of your web application pages, you should also offer the user the ability to "Add a new auction-item". The user should be presented with a form that reflects the data-model of an auction-item. Clicking on a submit button will then add the item into the database, and confirm to the user a simple message, such as: "Thanking you for adding an item".

5. Populating your Database

You should also submit along with this application, a populated database of a variety of Auction-Items. We would expect a minimum of 25 such items.

Note: You are at liberty as to whether you generate this XML dataset manually, or utilise your implemented Web-Application to generate this file.

6. The WishList

The WishList acts similar to a "shopping cart" for items that a user has been browsing. The header of your application should thus also contain a button to the "WishList". Once clicked, the user will be presented with a list of the items that they have selected to be stored into their wish-list. (Note: Again, you do not need to persist such data to the database. The database will only be storing/retrieving auction-items. Rather the WishList can again simply be stored into Session attributes. In the next assignment, you will be required to design a complete relational database with user entities. At that point, you will then need to persist the user's wish-list into the database, so that it will be available the next time the user logs in. However, in this assignment as there is no log-in to be implemented - information such as the WishList will only live as long as the active session is alive.)

At the WishList page, users can click on Auction-Items to see further details. They should also be able to "remove" any items from the WishList. If the user's WishList is empty, then an appropriate message such as "WishList is Empty" should be displayed to the user, followed by a "Back" button, returning the user to the previous page.
